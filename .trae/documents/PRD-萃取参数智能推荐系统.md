# 产品需求文档 (PRD) - 萃取参数智能推荐系统

## 1. 产品概述

### 1.1 产品背景
咖啡萃取参数（粉水比、水温、萃取时间）直接影响最终咖啡饮品的风味表现。传统方式依赖咖啡师的经验，缺乏数据驱动的科学依据。本系统旨在通过机器学习算法，基于历史萃取数据和用户评分，为不同豆种和烘焙度组合提供科学的萃取参数推荐。

### 1.2 产品目标
- 为用户提供精准的萃取参数推荐（粉水比、水温、时间区间）
- 建立数据采集和持续优化机制，不断提升推荐精度
- 通过可视化帮助用户理解推荐逻辑
- 确保系统高性能和高可用性

### 1.3 目标用户
- 家庭咖啡爱好者
- 专业咖啡师
- 咖啡馆经营者
- 咖啡豆烘焙商

## 2. 功能需求

### 2.1 数据采集模块
**功能描述**：收集存储不同豆种、烘焙度对应的历史萃取参数及用户评分数据
- 支持录入至少100种不同豆种与烘焙度组合
- 每条记录包含：豆种类型、品种、产地、处理法、烘焙程度、粉水比、萃取水温、萃取时间、用户综合评分（1-5分）、酸/甜/醇/余韵/平衡五维评分
- 数据存储：使用 Dexie(IndexedDB) 本地持久化

### 2.2 算法模型构建
**功能描述**：基于收集的数据训练推荐算法模型
- 采用加权回归模型分析参数与评分的相关性
- 计算各参数维度的最优值区间
- 基于豆种特征向量进行相似组匹配
- 采用贝叶斯平滑处理稀疏数据

### 2.3 推荐引擎
**功能描述**：根据输入输出推荐参数
- **输入参数**：豆种类型（必选）、烘焙程度（必选）
- **输出结果**：
  - 粉水比建议值（精确到小数点后一位，如 1:15.5）
  - 水温建议范围（精确到整数度，如 88-92℃）
  - 萃取时间区间建议（精确到分钟，如 2-3分钟）
  - 推荐置信度百分比
  - 推荐依据说明

### 2.4 用户交互界面
- 参数输入区：豆种选择器、烘焙度选择器、推荐触发按钮
- 结果展示区：三个核心参数卡片展示，置信度，依据说明
- 推荐依据面板：展示历史相似组合的数据统计
- 历史数据对比表格：展示Top 5最相似历史记录

### 2.5 系统优化机制
- 用户对每次推荐结果进行1-5星满意度评分
- 新评分数据自动纳入训练集
- 算法模型根据新数据自动更新权重
- 提供评分历史查看功能

### 2.6 数据可视化
- 参数-评分散点图：展示粉水比、水温、时间与综合评分的关系
- 烘焙度热力图：不同烘焙度下各参数的最优区间
- 推荐迭代趋势图：展示推荐满意度随时间的变化

## 3. 非功能需求

### 3.1 性能要求
- 推荐结果生成响应时间 ≤ 2秒
- 支持同时处理1000+并发请求（前端异步处理+Web Worker）
- 首屏加载时间 ≤ 3秒

### 3.2 质量要求
- 经过至少100种不同豆种与烘焙度组合的测试验证
- 推荐参数的用户满意度评分 ≥ 4.0分（5分制）
- 数据完整性校验机制

### 3.3 兼容性要求
- 支持主流浏览器（Chrome、Edge、Firefox、Safari）
- 响应式设计，支持桌面和移动端

## 4. 数据规范

### 4.1 豆种分类
| 类别 | 示例 |
|------|------|
| 阿拉比卡 | 耶加雪菲、瑰夏、哥伦比亚、曼特宁 |
| 罗布斯塔 | 越南罗布斯塔、印度罗布斯塔 |
| 利比里亚 | 利比里亚单品 |
| 混合豆 | 意式拼配、综合拼配 |

### 4.2 烘焙度分类
极浅烘焙、浅烘焙、中浅烘焙、中烘焙、中深烘焙、深烘焙、极深烘焙

### 4.3 参数取值范围
- 粉水比：1:8 ~ 1:18（手冲类）、1:1 ~ 1:5（意式类）
- 水温：80℃ ~ 100℃
- 萃取时间：0.5分钟 ~ 20分钟
